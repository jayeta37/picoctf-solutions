Pie Time Binary Exploitation - Easy

You’ve come across a binary using a security technique known as PIE (Position Independent Executable). Can you overcome it and find the flag?

PIE means the memory addresses within the binary are randomized each time it's run. This makes exploiting it trickier, but not impossible.

You can leak addresses at runtime using debugging tools like GDB. Once you know the base address, you can calculate the locations of functions.

Run checksec on the binary:
$ checksec pie_time
You’ll see that PIE is enabled if the binary is position independent.

Use GDB with pwndbg or gef to analyze memory, leak addresses, and construct your exploit. You may need to calculate the offset for your payload.
Solution:

1. Run checksec pie_time to confirm PIE is enabled.
2. Open the binary with GDB:

$ gdb pie_time
3. Use a format string vulnerability to leak an address at runtime.
4. Calculate the PIE base by subtracting known offsets from the leaked address.
5. With the base address, compute the location of useful functions or flags.
6. Construct your exploit payload accordingly.
7. Upon successful exploitation, the binary will reveal the flag.

✅ The flag is revealed after exploiting the PIE-protected binary using leaked addresses.
